# 2026-03-25 — Phase 2.4: vision-mcp

## Summary

Implemented `vision-mcp` — the visual perception layer for AI-HIL — using a Logitech MX Brio Ultra 4K USB camera.

## Hardware

- **Camera:** Logitech MX Brio Ultra 4K (UVC, VID=0x046d PID=0x0944)
- **Connection:** USB, detected at camera index 0 on macOS
- **Native resolution reported by OpenCV:** 1920×1080 (AVFoundation backend)

## Architecture Decisions

### Why not hardware UVC controls via pyusb?

macOS kernel (AppleUSBVideoSupport) claims Interface 0 (VideoControl, class 0x0e subclass 0x01) and holds it while the camera is active. `pyusb` control transfers return empty — the kernel driver intercepts them. Detaching the driver would break OpenCV capture.

### Why not direct AVFoundation for all controls?

Without an active `AVCaptureSession`, AVFoundation returns 0 for all camera properties (exposure, WB, lensPosition, zoom). Standing up a persistent session in Python conflicts with OpenCV's own capture. For most use cases, post-capture OpenCV transforms produce equivalent results without session management complexity.

### Software ePTZ instead of hardware PTZ

The MX Brio's pan/tilt is electronic (crops the 4K sensor). AVFoundation does not expose the 4K preset (`supportsAVCaptureSessionPreset(3840x2160) = false`) and `maxAvailableVideoZoomFactor = 1.0`, so no zoom range is available via AVFoundation. Software crop at 1080p gives ~2× effective range before quality loss — sufficient for board inspection.

### LED detection: OpenCV-first, Claude vision fallback

OpenCV HSV thresholding is fast, offline, and deterministic for well-lit LED blobs. Confidence is computed from saturation + blob area. If confidence < 60% (poor lighting, unusual colors, cluttered scene) and `ANTHROPIC_API_KEY` is set, the same frame is sent to `claude-opus-4-6` for interpretation.

## Known Limitations

- True hardware ePTZ (full Logitech pan/tilt range) requires the Logitech Vendor Specific interface (Interface 3, class 0xff) — protocol is proprietary
- `set_focus` uses AVFoundation and requires a brief exclusive lock on the device; may conflict if OpenCV capture is simultaneous
- Camera permission must be granted to the terminal process in macOS System Settings → Privacy & Security → Camera
