### `jtag-mcp-rs` — Active Debugging Capabilities

Added 6 new tools and expanded `read_registers`:

#### New tools

| Tool | Description |
|------|-------------|
| `reset_target` | `reset_and_halt`, reads post-reset PC, resumes — soft reset without reflash |
| `write_memory(address, data)` | Halts CPU, writes hex bytes to any address, resumes — enables fault injection and live variable patching |
| `set_breakpoint(address)` | Sets hardware breakpoint via FPB unit (`probe-rs` `set_hw_breakpoint`) — CPU halts when PC reaches address |
| `clear_breakpoint(address)` | Clears FPB breakpoint at address |
| `set_watchpoint(address, kind)` | Sets DWT data watchpoint; `kind` = `"read"` / `"write"` / `"read_write"` (default `"write"`) |
| `clear_watchpoint(address)` | Zeros DWT FUNCTION register for matching comparator unit |

#### `read_registers` — added xPSR and EXTRA

Synced with Python `jtag-mcp`: now also reads:
- `xPSR` — full Program Status Register
- `EXTRA` — probe-rs packed register: `CONTROL[31:24] | FAULTMASK[23:16] | BASEPRI[15:8] | PRIMASK[7:0]`

#### Watchpoint implementation note

probe-rs 0.31.0 has no high-level watchpoint API. Watchpoints are implemented by directly
writing to memory-mapped DWT registers:

```
DEMCR (0xE000EDFC)    — bit 24 TRCENA must be set before DWT is accessible
DWT_CTRL (0xE0001000) — bits [31:28] = NUMCOMP (number of comparators)
DWT_COMPn (0xE0001020 + n*0x10) — comparator address
DWT_MASKn (0xE0001024 + n*0x10) — 0 = exact address match
DWT_FUNCn (0xE0001028 + n*0x10) — 0b0101=read, 0b0110=write, 0b0111=read_write
```

`set_watchpoint` enables TRCENA, reads NUMCOMP live from DWT_CTRL, scans for a free
comparator (FUNCTION[3:0] == 0), then writes COMP/MASK/FUNCTION.
`clear_watchpoint` scans all comparators for the matching address and zeros FUNCTION.

STM32WL55 CM4 has 4 DWT comparator units.
