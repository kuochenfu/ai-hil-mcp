#### `read_registers` ‚Äî added xPSR and EXTRA

Synced with Python `jtag-mcp`: now also reads:
- `xPSR` ‚Äî full Program Status Register
- `EXTRA` ‚Äî probe-rs packed register: `CONTROL[31:24] | FAULTMASK[23:16] | BASEPRI[15:8] | PRIMASK[7:0]`

#### Watchpoint implementation note

probe-rs 0.31.0 has no high-level watchpoint API. Watchpoints are implemented by directly
writing to memory-mapped DWT registers:

```
DEMCR (0xE000EDFC)    ‚Äî bit 24 TRCENA must be set before DWT is accessible
DWT_CTRL (0xE0001000) ‚Äî bits [31:28] = NUMCOMP (number of comparators)
DWT_COMPn (0xE0001020 + n*0x10) ‚Äî comparator address
DWT_MASKn (0xE0001024 + n*0x10) ‚Äî 0 = exact address match
DWT_FUNCn (0xE0001028 + n*0x10) ‚Äî 0b0101=read, 0b0110=write, 0b0111=read_write
```

`set_watchpoint` enables TRCENA, reads NUMCOMP live from DWT_CTRL, scans for a free
comparator (FUNCTION[3:0] == 0), then writes COMP/MASK/FUNCTION.
`clear_watchpoint` scans all comparators for the matching address and zeros FUNCTION.

STM32WL55 CM4 has 4 DWT comparator units.

#### write_memory test note

First attempt at `0x20003F00` returned stale data on readback ‚Äî that address is inside the
active stack region (SP=`0x20003EC0`, stack grows down). Firmware overwrote the bytes during
the brief window between write and read. Second attempt at `0x20001000` (mid-SRAM, between
BSS end and stack) persisted correctly.

#### clear_breakpoint bug + fix

`do_clear_breakpoint` called `clear_hw_breakpoint` without halting first. probe-rs rejects
the FPB config write when the CPU is in breakpoint-halt state (halt-by-FPB vs halt-by-debug-
request are different states). Fix: added `core.halt()` + `core.run()` around the clear call,
matching the pattern used by `set_watchpoint` and `write_memory`.
